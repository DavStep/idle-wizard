# Top Panel

The top panel is shared room UI. It shows player name, mana, coin, and crystal above each page.

Use the compact player-card composition: the avatar spans the left side and stays grounded to the panel edge; username and currencies share the first row; mana and its `+n/s` rate share the second row; and the compact white/purple level badge sits beside one continuous purple request rail. In this panel, resource icons lead their values. Keep mana subordinate to the username and preserve visible vertical separation between username, mana, and the level badge.

The quest rail uses a single 1px outer stroke, a real 2px gap between the stroke and fill, and a flat solid-purple fill with no gradient, highlight, or inner shadow. Its ends are rounded rectangles with short straight side sections, not full pill caps. Thin ticks divide the rail into one segment per configured request plus the final coin payment. Keep the small `Complete N more quests to level up` caption left-aligned directly under the rail with bottom breathing room, without letting the caption shift the rail away from the badge's optical center. Completed requests send a short three-star flight into the newly completed segment, except when reduced motion is requested. Shrink resource text only when it would clip. Mana uses the same vivid blue token everywhere.

The rail frame is drawn directly in CSS. It is simple flat geometry, so rasterizing it into a nine-slice only adds resampling and corner distortion. Keep the frame, fill, and divider state as separate DOM layers. Reveal the fill with clipping instead of `scaleX`, which would squash its rounded leading corners. The divider immediately after a completed segment is dark; future dividers remain light.

Open the reproducible non-persistent `1 / 4` review state with `/?devUi=topPanelQuestProgress` or `cheats.openUi('topPanelQuestProgress')`. The preview hides the tutorial backdrop without changing tutorial progress, so native-pixel captures preserve the panel's real colors.

Settings contain username editing, the old report tab for feedback/bugs/features, and the configurations tab for device toggles, visual theme, font, and progress bar. Pressing the top-panel avatar opens the avatar picker directly; the picker shows avatar tiles directly in the dialog, with locked choices as muted silhouettes. Device toggles such as haptics, music, and sfx stay device-local. Visual option names select researched options only. The right-side price/status label, including `free`, researches unresearched options; after that the name can select the option. The avatar, existing resource icons, white/purple level badge, and continuous segmented purple rail are the owned cartoony exceptions inside otherwise quiet top chrome.
